# 2025-01-29 Moltbot Documentation Research

## Key Findings

Today I researched comprehensive Moltbot documentation covering:

1. **Core Configuration** - Location at ~/.clawdbot/moltbot.json, supports JSON5 format with various sections for agents, channels, models, etc.

2. **Model Management** - Supports multiple providers (Anthropic, OpenAI, Google/Gemini, etc.), with fallback mechanisms and aliasing system. Models can be switched via CLI or in-chat commands.

3. **Webhooks** - Built-in HTTP webhook endpoints for external triggering, with authentication via shared tokens.

4. **Cron Jobs** - Built-in scheduler for persistent job management, with both main session and isolated execution modes.

5. **Security Features** - Multiple layers including identity verification, scope limitation, and model safeguards. Includes security audit functionality.

6. **Session Management** - Flexible session scoping with daily and idle reset policies, supporting multiple channels and users.

7. **Tool System** - Extensive tool framework with allow/deny lists, profiles, and grouped tool categories.

8. **Skills Framework** - AgentSkills-compatible skill system with multiple loading locations and precedence rules.

9. **Memory System** - Markdown-based memory with vector search capabilities and automatic indexing.

10. **Heartbeat Mechanism** - Periodic agent turns for background monitoring and notifications.

## Configuration Structure

The main configuration file (~/.clawdbot/moltbot.json) follows this structure:
- agents.defaults - Default agent settings
- channels - Channel-specific configurations (WhatsApp, Telegram, Discord, etc.)
- models - Model configuration and provider settings
- tools - Tool access control and grouping
- gateway - Network and authentication settings
- hooks - Webhook configuration
- cron - Scheduled job configuration
- skills - Skill loading and management

## Installation and Setup

Moltbot can be installed via:
- curl installer script (recommended)
- npm global installation
- From source for development

The onboard command provides guided setup for models, gateway, channels, and security.

## Security Considerations

The security model emphasizes:
- Identity-first approach (who can talk to the bot)
- Scope limitations (where the bot is allowed to act)
- Tool restrictions and sandboxing
- Regular security audits
